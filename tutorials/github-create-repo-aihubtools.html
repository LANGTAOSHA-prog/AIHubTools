GitHub 新建仓库完整教程：从注册到创建 AIHubTools

这篇教程是写给第一次使用 GitHub 的朋友看的。你不需要懂代码，也不用先学一堆专业名词，只要跟着图片一步一步操作，就能把自己的 AIHubTools 仓库创建出来。仓库可以理解成一个放项目文件的在线文件夹，后面你的网站首页、教程页、图片、工具页面，都可以慢慢放进去。

适合：完全新手 设备：电脑浏览器 仓库名：AIHubTools 路径适配：GitHub Pages

教程目录

  • 注册或登录 GitHub 账号
  • 从首页进入 New repository
  • 填写仓库名、公开状态和基础选项
  • 创建成功后认识仓库首页
  • 后续如何创建文件、上传网站内容

一、先注册或登录 GitHub

如果你还没有 GitHub 账号，进入 GitHub 后会看到注册页面。页面上可以选择使用邮箱注册，也可以选择 Continue with Google 或 Continue with Apple。对新手来说，用常用邮箱注册比较直观；如果你平时习惯 Apple 账号，也可以用 Apple 登录。

图片 1：注册 GitHub 账号，填写 Email、Password、Username 和国家地区。

注册时最重要的是三个内容：邮箱、密码、用户名。用户名以后会出现在你的仓库链接里，所以建议简单一点，不要太长。国家地区按照实际情况选择即可。填写完成后点击 Create account，按照提示完成验证。

图片 2：如果选择 Apple 登录，会跳转到 Apple 账号登录页面。
图片 3：输入 Apple ID 后继续确认登录。
图片 4：如果开启了双重认证，需要输入手机或设备收到的验证码。
图片 5：如果是自己的电脑，可以选择“信任”；公共电脑建议选择“不信任”。
新手提醒：如果你在网吧、公司共用电脑或临时设备上登录，不建议选择“信任此浏览器”。自己的电脑可以信任，这样下次登录会少一步验证码。

二、从首页进入新建仓库

登录成功后，你会进入 GitHub 的 Dashboard 首页。页面左侧通常会显示已有仓库，右上角会有一个加号按钮。点击加号后，会出现下拉菜单，里面有 New issue、New repository、Import repository 等选项。这里要选择 New repository，也就是新建仓库。

图片 6：在 GitHub 首页点击右上角加号，然后选择 New repository。
注意：New issue 是新建议题，不是创建项目；Import repository 是导入已有仓库。我们现在要创建全新的 AIHubTools，所以选择 New repository。

三、填写 AIHubTools 仓库信息

进入 Create a new repository 页面后，先看 Owner 是否是你的账号。然后在 Repository name 里填写 AIHubTools。这个名字最好固定，不要一会儿写 AiHubtools，一会儿写 AIHubTools，因为后续写教程、配置 GitHub Pages、引用链接时，统一名称会省很多麻烦。

图片 7：填写仓库名称 AIHubTools，选择 Public，按需要设置 README、.gitignore 和 License。

确认 Owner

Owner 就是仓库属于谁。个人项目一般选择自己的账号。如果你有多个组织或账号，创建前一定看一眼，避免建错位置。

填写 Repository name

在仓库名输入框中填写 AIHubTools。如果下面出现 available，说明这个名字可以使用。

选择 Public

如果你准备用 GitHub Pages 做公开网站，建议选择 Public。Private 是私有仓库，不适合普通公开网站教程演示。

README、.gitignore、License 怎么选

新手可以先不勾选 README，也可以后面再创建。License 如果暂时不确定，可以先选 No license。等项目内容稳定后再补也可以。

点击 Create repository

确认仓库名、公开状态没有问题后，点击绿色的 Create repository 按钮。点完之后，GitHub 会自动跳转到新仓库页面。

四、创建成功后的仓库页面

仓库创建成功后，你会看到 AIHubTools 的仓库首页。因为现在仓库还是空的，所以页面会显示 Quick setup，并提供 creating a new file 和 uploading an existing file 两个入口。这里不用紧张，看到这个页面就说明仓库已经创建成功了。

图片 8：AIHubTools 仓库创建成功，出现 Quick setup 页面。
判断是否成功：页面左上角能看到你的用户名 / AIHubTools，并且仓库名旁边显示 Public，就说明仓库已经建好了。

五、后续如何创建第一个文件

如果你想先测试仓库是否能正常保存文件，可以点击 creating a new file。进入新建文件页面后，上方填写文件名，中间输入文件内容，右上角或下方会看到 Commit changes。提交后，这个文件就会保存到仓库里。

图片 9：创建新文件页面，可以填写文件名和内容，最后提交保存。

比如后续你要做网站首页，就可以创建 index.html；要放教程，就可以创建 tutorials/xxx.html；要放图片，就可以放到 images/tutorials/ 目录。GitHub 里输入带斜杠的文件名，例如 tutorials/github-guide.html，会自动生成对应文件夹。

六、后续图片和页面怎么放

为了让 AIHubTools 项目后期更好管理，建议从一开始就按目录整理文件。教程页面统一放在 tutorials/，教程图片统一放在 images/tutorials/ 或更细的子目录里。这样以后你写 10 篇、20 篇教程，也不会乱。

写给第一次创建仓库的你

第一次用 GitHub 时，看见很多英文按钮很正常，不要因为看不懂每个单词就停下来。你只需要记住这条主线：先注册或登录账号，再点击 New repository，然后填写仓库名 AIHubTools，选择 Public，最后点击 Create repository。仓库建好以后，它就是你这个项目的基础位置。

后面你要做的事情，都是围绕这个仓库继续展开：上传首页、添加工具页面、放教程文章、插入图片、配置 GitHub Pages。只要第一步仓库创建正确，后面的工作就会顺很多。建议你创建完成后，把仓库地址收藏起来，后面每次修改网站都从这个仓库进入。
