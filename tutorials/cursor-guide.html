一、什么是 Cursor？它与 VS Code + Copilot 有何不同？

Cursor 是一款专为 AI 协作设计的代码编辑器，它基于 VS Code 的开源版本二次开发，因此完美继承了 VS Code 的生态和使用习惯。你可以把它理解为“一个深度整合了多种强大 AI 能力的 VS Code”。

与 GitHub Copilot 主要提供代码补全不同，Cursor 的核心优势在于能理解整个代码库的上下文。这意味着它可以帮你完成更复杂的任务，比如跨文件重构、回答项目架构问题、或者根据需求直接生成新功能模块的代码。

二、安装与配置

访问 Cursor 官网 cursor.sh 下载对应操作系统的版本。安装后，Cursor 会引导你一键从现有的 VS Code 中导入所有扩展和设置，整个过程无缝衔接。

免费版每月提供一定数量的快速 AI 功能调用次数和少数几次使用 GPT-4/Claude 3 Opus 等顶级模型的机会，对于轻度用户基本足够。

三、三大核心 AI 功能

掌握以下三个快捷键，你就掌握了 Cursor 80% 的核心用法：

  • Tab（代码补全）：与 Copilot 类似，在你输入代码时，它会自动提供代码补全建议。
  • Cmd/Ctrl + K（内联编辑/生成）：选中一段代码，按下 `Cmd + K`，然后用自然语言描述你的修改需求（如“将这段代码转换为 React Hooks 写法”或“为这个函数增加错误处理”），AI 会直接在原地生成修改建议。
  • Cmd/Ctrl + L（AI 聊天）：打开一个聊天窗口，你可以像和 ChatGPT 对话一样，向 AI 提问。关键在于，这个聊天窗口能理解你当前打开的文件乃至整个项目。

四、进阶功能：Codebase 与 Composer

@Codebase：与你的整个项目对话

在聊天窗口中输入 `@Codebase`，Cursor 就会索引你当前项目的全部文件。之后，你可以提出关于整个项目的宏观问题，例如：

@Codebase 项目的用户认证流程是怎样的？涉及到哪些文件？
@Codebase 我想新增一个处理支付的 API 路由，应该在哪个文件里添加？

Composer：跨文件重构利器

按下 `Cmd/Ctrl + Shift + I` 可以打开 Composer 视图。这里允许你下达需要修改多个文件的大型重构指令。例如：“将项目中所有使用 class 组件的 React 文件都改为 function 组件和 Hooks。” Cursor 会分析并列出所有需要修改的地方，生成 diff 预览，待你确认后一键应用。

五、高效使用技巧

  • 自定义规则 (`.cursorrules`)：在项目根目录创建一个 `.cursorrules` 文件，里面可以写下项目的编码规范、使用的技术栈、代码风格等。Cursor 在生成或修改代码时会严格遵守这些规则。
  • 精准 `@` 引用：在聊天中，除了 `@Codebase`，你还可以用 `@` 精准引用某个文件（`@src/utils.ts`）、某个函数（`@someFunction`）甚至某个文档（`@README.md`），让 AI 的回答更加聚焦和准确。

六、常见问题与解答 (FAQ)

Cursor 和 GitHub Copilot 有什么区别？
Copilot 主要是一个强大的“代码补全”工具，而 Cursor 更像一个“AI 结对程序员”。Cursor 整合了聊天、代码生成、代码编辑和全库问答等多种能力，能够处理比 Copilot 更复杂、更宏观的编程任务。
Cursor 是免费的吗？
Cursor 提供带有额度限制的免费版和付费的 Pro/Business 版。免费版足以满足轻度使用和体验，而 Pro 版则提供更多对 GPT-4 和 Claude 3 Opus 等先进模型的调用次数，以及更快的 AI 响应速度。
我原来的 VS Code 配置怎么办？
无需担心。Cursor 首次启动时会提供一个“导入”选项，可以一键将你 VS Code 中的所有扩展、主题、设置和快捷键全部迁移过来，实现无缝切换。
@Codebase 功能的隐私安全吗？
根据 Cursor 官方文档，Codebase 功能通过在本地将你的代码转换为向量嵌入，然后将相关代码片段发送给 AI 模型进行处理，你的完整代码库不会被上传到任何服务器。
